目前启动的命令，文件名称，路径相对固定

当前方法适合在雨云的mcsm里面，ttyd终端一直运行，all.sh只作为一个启动管理软件。

MCSM 游戏容器自带默认启动脚本，容器启动时会自动执行该脚本；我们只需修改这个默认启动脚本，就能自定义要运行的程序。

目前脚本逻辑：容器启动脚本会拉起 TTYD 网页在线终端，用户通过浏览器接入终端后，再手动执行 all 脚本。all 脚本支持按不同目录批量启动多个 Java 程序，当前启动程序列表为固定写死配置。

all.sh脚本对比说明

## 一、两个版本的区别

| 对比项            | 原版（你上传的）          | v5 版（现在用的 [all.sh](all.sh)）      |
| ----------------- | ------------------------- | --------------------------------------- |
| **配置方式**      | JSON 格式，Python3 解析   | Bash 数组，纯 Bash 解析                 |
| **依赖**          | 需要 Python3              | 纯 Bash，无外部依赖                     |
| **MCMS 能运行吗** | ❌ 不能（MCMS 没 Python3） | ✅ 可以                                  |
| **严格模式**      | set -e（出错就退出）      | set +e（容错，不容易意外退出）          |
| **清屏方式**      | /usr/bin/clear            | 混合清屏（空行 + 光标上移），菜单在顶部 |

## 二、版新增的功能

1. **全局 stop 响应** - 任何界面（主菜单、子菜单、看日志、控制台）都能响应面板的停止按钮
2. **统一返回指令** - 只用 `0` 返回上一级，`00` 直接回主菜单，删掉了 q/quit/exit 等其他方式
3. **服务自动发现** - 添加服务只要复制一个配置块，自动识别，不用手动改列表
4. **自启动自动执行** - 脚本启动时自动启动所有标记为自启动的服务
5. **自启动存脚本本体** - 所有配置都在一个文件里，改自启动直接改脚本本身
6. **查看日志显示状态** - 日志菜单也显示服务运行中 / 已停止
7. **0 退出停止所有服务** - 主菜单输入 0 退出时，先停掉所有服务再退出

## 三、MCMS 面板注意事项

1. **MCMS 环境没有 Python3** → 原版跑不了，必须纯 Bash
2. **停止按钮发的是 stop 字符串** → 不是信号，脚本要主动读 stdin 检查
3. **TERM 变量可能没设置** → clear 命令会报错，严格模式下直接退出
4. **启动脚本不能删 / 改名** → `启动脚本(可修改).sh` 是容器入口，删了容器起不来

## 四、配置格式

v5 版配置（每个服务一个块，复制就能加新服务）：

```bash
SERVICE_1=(
  'name: "zenith-bin"'               # 服务名
  'exec_file: "ZenithProxy"'          # 可执行文件名
  'work_dir: "ZenithProxy"'           # 工作目录
  'exec_type: ""'              # 运行类型（留空自动识别）
  'exec_command: ""'           # 自定义启动命令（留空自动生成）
  'auto_start: "false"'           # 是否自启动
)
```

## 五、菜单操作

| 操作               | 指令         |
| ------------------ | ------------ |
| 启动服务           | 主菜单按 `1` |
| 停止服务           | 主菜单按 `2` |
| 自启动管理         | 主菜单按 `3` |
| 查看日志           | 主菜单按 `4` |
| 进入控制台         | 主菜单按 `5` |
| 返回上一级         | 按 `0`       |
| 直接回主菜单       | 按 `00`      |
| 退出（停所有服务） | 主菜单按 `0` |
