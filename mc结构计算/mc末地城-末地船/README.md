# 末地飞船查找工具（End Ship Finder）

一款独立 Python 工具，用于查找《我的世界》Java版1.20版本的末地城与末地飞船。

本工具不导入、不关联任何原生C语言项目。所需的Java版随机数生成算法（RNG）、末地生物群系噪声、末地城地形检测、末地城结构生成逻辑，均已在 `end_finder.py` 文件中独立实现。

## 使用方法

查找指定坐标附近的末地城：

```powershell
python .\end_finder.py --seed -6975987475394236399 --x 23704 --z -3816 --radius 1000
```

仅显示生成了末地飞船的末地城：

```powershell
python .\end_finder.py --seed -6975987475394236399 --x 23704 --z -3816 --radius 1000 --ships-only
```

### 输出字段说明

- `city_x`、`city_z`：末地城结构的坐标位置
- `distance`：与搜索中心坐标的直线距离
- `biome`：该结构检测判定对应的末地生物群系
- `terrain_height`：关键角落最低地形高度（末地城生成要求该数值至少为60）
- `has_ship`：该末地城是否生成了末地飞船结构
- `ship_x`、`ship_y`、`ship_z`：存在末地飞船时，飞船主体的基准坐标
- `pieces`：结构生成器生成的末地城组件数量

## 当前功能范围

本工具功能做了精简限定，仅针对《我的世界》Java版1.20版本的末地城、末地飞船进行检索，暂未实现其他维度、其他结构的查找功能。

## 迁移与算法注意事项

### 1. 结构尝试点不等于实际末地城

最开始容易犯的错误是：只根据末地城的区域随机算法算出 `city_x`、`city_z`，然后把这些坐标直接当作“末地城位置”。

这是不完整的。末地城的查找至少要经过三层判断：

1. **结构区域尝试点**：每个末地城区域大小为 `20 x 20` 个区块，salt 为 `10387313`，在区域内随机出一个结构尝试位置。
2. **末地生物群系检查**：该尝试点对应的末地生物群系必须是 `end_midlands` 或 `end_highlands`。
3. **末地地形高度检查**：末地城要求关键角落的最低地形高度至少为 `60`。

如果只做第1步，会得到大量假阳性坐标；这些坐标看起来像末地城候选点，但游戏里不一定真的生成末地城。

### 2. 末地飞船不是独立结构

末地飞船不是一个像村庄、神殿那样可以单独用区域算法定位的结构。它是末地城结构生成过程中的一个组件，也就是 End City piece。

因此查找末地飞船的正确流程是：

1. 找到真实可生成的末地城。
2. 对该末地城运行 End City piece 生成逻辑。
3. 在生成出的 pieces 中查找类型为 `END_SHIP` 的组件。

本工具输出的 `ship_x`、`ship_y`、`ship_z` 是飞船 piece 的基准坐标，不是通过独立结构定位算法算出来的。

### 3. 必须复刻 Java Random 的细节

迁移到其他语言时，Java版随机数必须严格复刻：

- `setSeed(seed)` 使用 `(seed ^ 0x5DEECE66D) & ((1 << 48) - 1)`。
- 每次 `next(bits)` 使用 `seed = (seed * 0x5DEECE66D + 0xB) & ((1 << 48) - 1)`。
- `nextInt(n)` 对 2 的幂和非 2 的幂有不同逻辑。
- `nextLong()` 要注意 Java 有符号 32 位拼接的行为。

如果这些细节不一致，末地城位置、地形旋转、pieces 生成、是否有船都会偏掉。

### 4. 坐标含义要区分清楚

输出中的 `city_x`、`city_z` 是末地城结构尝试位置，通常是结构所在区块相关的定位点；实际建筑主体可能会围绕这个位置展开，并不代表玩家脚下精确就是建筑中心。

输出中的 `ship_x`、`ship_y`、`ship_z` 是飞船组件的 piece 基准位置。实际船体也有尺寸和朝向，进入游戏后可以在该坐标附近寻找。

### 5. 版本范围暂时只保证 Java 1.20

当前实现按 Java 1.20 的行为整理。虽然部分算法在相邻版本中可能相同，但迁移或扩展到其他版本时需要重新核对：

- 末地城结构配置是否变化。
- 末地生物群系生成是否变化。
- 地形高度/噪声采样是否变化。
- End City pieces 的递归生成逻辑是否变化。

在没有确认前，不建议把本工具直接当作所有 Java 版本通用实现。
